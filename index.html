HTML

Hyper Text Markup Language (HTML for short) is the standard markup language for documents designed to be displayed in a web browser. It defines the content and structure of web content. It is often assisted by technologies such as Cascading Style Sheets (CSS) and scripting languages such as JavaScript.

HTML provides:

  • The text you read
  • The layout of the webpage
  • Accesibily features for screen readers

CSS

Cascading Style Sheets (CSS for short) is a stylesheet language used specifying the presentation and styling of a document written in HTML or XML. CSS is a cornerstone technology of the World Wide Web, alongside HTML and JavaScript.

CSS provides:

  • Background colors
  • Font styles
  • Simplified Webpage styling
  • Site-wide consistancy

Web Page Content

HTML uses elements to tell browsers what goes where. HTML documents start with a !DOCTYPE html element to let the browser know what language the document is written in. You can create a title using the h1 element, which sets the text inside to the largest font size to show its importance. Webpages usually only have one of these.

p elements are used for paragraphs. A webpage can have as many p elements as is necessary. Other things HTML can do is make tables, buttons, form fields, and provide accessibility features for the visually impaired.

Styling

As stated, styling is done using CSS. This is acheived by linking a stylesheet that has all the styling code you've created to an HTML document using the link element. In your stylesheet, you select the HTML elements that you want to style either by naming them directly, or by assigning a class or id to multiple elements that you want to group together.

You can also use the same stylesheet accross a website so that every page is uniform; only having to make minor changes to account for different elements.

Accessibility

Both HTML and CSS work together using specific elements and styling choices to provide a web page that is accessible to all. It is best practice to include tags for screen readers and to take visual impairment into account when building a web site.

Some scroll types can cause discomfort for those with certain visual disabilities. For this reason, it is best to include a smooth scroll style.
